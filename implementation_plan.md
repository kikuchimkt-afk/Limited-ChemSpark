# 究極の化学共通テスト 正誤問題アプリ設計

## ゴール

24章 × 50問 = **1,200問** の正誤四択問題を、音声読み上げで学習できるWebアプリを作成する。

---

## 制約条件

| 条件 | 理由 |
|---|---|
| 数式を使用しない | 音声読み上げ対応 |
| 化学反応式を使用しない | 音声読み上げ対応 |
| 四択形式 | 共通テスト形式 |
| 「重箱の隅つつき」級の紛らわしさ | 正確な知識を鍛える |

---

## 問題生成の実現可能性

### データの活用ポイント

json_dataの各エントリから以下のような「ひっかけ」を自動的に設計できます：

| データ要素 | 出題のネタ | ひっかけの方向性 |
|---|---|---|
| `substances[].properties.color` | 「～は無色である」 | 色を微妙に変える（淡青→無色） |
| `substances[].properties.solubility` | 「～は水に溶けやすい」 | 溶けやすい/溶けにくいを入れ替え |
| `concepts[].definition` | 定義の正誤判定 | キーワードを1語だけ変える |
| `concepts[].conditions` | 成立条件の正誤 | 条件を微妙に変える |
| `concepts[].counter_examples` | 混同しやすい概念 | 同素体↔同位体 を入れ替え |
| `concepts[].mnemonics` | 語呂合わせの正確さ | 元素記号を1つ変える |
| `concepts[].examples` | 具体例の正誤 | 別の物質をまぜる |
| `relations[].relation_type` | 概念間の関係 | 前提関係を逆にする |
| `substances[].type` | 物質の分類 | 単体↔化合物を入れ替え |
| `concepts[].steps` | 実験手順 | 手順の順序や理由を変える |

### 各章50問の内訳案

1章あたりの **推奨出題バランス**：

| 問題カテゴリ | 問数 | 説明 |
|---|---|---|
| 🔵 定義の正誤 | 12問 | 「～とは…である」の正誤 |
| 🟢 物質の性質 | 10問 | 色・融点・溶解度・導電性など |
| 🟡 分類・判別 | 8問 | 単体/化合物/混合物、同素体の判別 |
| 🟠 実験手順・操作 | 6問 | 「ろ過では～する」の正誤 |
| 🔴 ひっかけ混同 | 8問 | 類似概念の取り違え |
| 🟣 語呂合わせ確認 | 3問 | 暗記の正確さチェック |
| ⚫ 総合・応用 | 3問 | 複数概念の組合せ |

## 出題パターン詳細（音声読み上げ対応）

### パターン1: 定義の一語すり替え（最もひっかけやすい）

```
Q: 同素体について、正しい記述を選びなさい。

① 同じ元素の単体であるが、原子の数や配列が異なるため性質が異なる物質群である
② 同じ元素の化合物であるが、原子の数や配列が異なるため性質が異なる物質群である ← 化合物→単体
③ 異なる元素の単体であるが、原子の数や配列が似ているため性質が似ている物質群である
④ 同じ元素の単体であるが、中性子の数が異なるため性質が異なる物質群である ← 同位体の定義
```

> [!TIP]
> **ひっかけのコツ**: 定義文の中の1～2語だけを変え、他は完全に正しくする。音声で聴くと違いに気づきにくい。

### パターン2: 物質の性質入れ替え

```
Q: オゾンについて、誤りを含む記述を選びなさい。

① 分子式はO3で表される酸素の同素体である
② 淡青色で独特のにおいがある
③ 反応しやすく有毒である
④ 常温で最も安定な酸素の単体である ← これは酸素O2の説明
```

### パターン3: 実験操作の理由すり替え

```
Q: 蒸留の操作について、正しい記述を選びなさい。

① 沸騰石を入れるのは、突沸を防ぐためである
② 温度計の球部を枝の付け根付近にするのは、液体の温度を正確に測るためである ← 「気体の温度」が正しい
③ フラスコ内の液量を半分以下にするのは、突沸で溶液が冷却器に流れ込むのを防ぐためである
④ 冷却水を下から上に流すのは、冷却器の中に空気が残らないようにするためである
```

### パターン4: 数値の微妙なすり替え（数式なし）

```
Q: 炎色反応の色について、正しい組合せを選びなさい。

① ナトリウムは黄色、カリウムは赤紫色
② リチウムは赤色、バリウムは青緑色 ← バリウムは黄緑
③ カルシウムは橙赤色、ストロンチウムは黄緑色 ← Srは紅
④ 銅は赤紫色、リチウムは黄色 ← 全部違う
```

### パターン5: 概念の混同（共通テスト頻出）

```
Q: 化学平衡について、正しい記述を選びなさい。

① 平衡状態では、正反応と逆反応の速度が等しくなっている
② 触媒を加えると平衡定数の値が変化する ← 変化しない
③ 温度を変えても平衡定数は変化しない ← 変化する
④ 平衡状態では分子の反応が完全に停止している ← 動的平衡
```

## 音声読み上げの実装

### Web Speech API（ブラウザ内蔵・無料）

```javascript
// 基本実装
const speak = (text) => {
  const utterance = new SpeechSynthesisUtterance(text);
  utterance.lang = 'ja-JP';
  utterance.rate = 0.9;  // やや遅め
  speechSynthesis.speak(utterance);
};
```

### 読み上げ順序
1. 問題文 → 2秒ポーズ
2. 「選択肢1」→ ①の内容 → 1秒ポーズ
3. 「選択肢2」→ ②の内容 → 1秒ポーズ
4. (...繰り返し)
5. 解答後: 解説文を読み上げ

### 化学用語の読み上げ対策

| 問題 | 対策 |
|---|---|
| 「H2O」は読めない | → データ作成時に「水」と記載 |
| 「NaCl」は読めない | → 「塩化ナトリウム」と記載 |
| 「℃」の読み方 | → 「度」に変換 |
| 「mol」の読み方 | → 「モル」に変換 |

> [!IMPORTANT]
> 問題文・選択肢にはすべて**日本語の読み仮名で記載**し、数式・化学式は一切含めないルールを徹底。

## 問題作成の進め方

### 方法A: AIによる一括生成（推奨）

1. json_dataの各ファイルを読み込み
2. Gemini/Claude APIで50問ずつ生成
3. 出題スキーマに沿ったJSONを出力
4. レビュー後にアプリに組み込み

### 方法B: テンプレートベース半自動生成

1. 出題パターンのテンプレートを10種類用意
2. json_dataの`concepts`と`substances`を当てはめ
3. 選択肢のひっかけも自動生成
4. 人間がレビュー・調整

## ひっかけパターンの分類（trap_type）

| trap_type | 説明 | 難易度 |
|---|---|---|
| `keyword_swap` | キーワード1語をすり替え | ★★★ |
| `concept_confusion` | 類似概念の混同 | ★★★★ |
| `property_swap` | 物質の性質を別物質と入れ替え | ★★★ |
| `condition_change` | 成立条件の変更 | ★★★★ |
| `step_order` | 手順の順序入れ替え | ★★ |
| `reason_swap` | 正しい操作に誤った理由をつける | ★★★★★ |
| `partial_truth` | 前半は正しく後半だけ誤り | ★★★★★ |
| `multiple_swap` | 複数箇所を同時にすり替え | ★★ |
| `allotrope_mix` | 同素体の性質を入れ替え | ★★★ |
| `negation` | 「～ない」の有無を変える | ★★★ |
